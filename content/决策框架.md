# 决策框架

## 我的决策原则

### 1. 先算最坏情况

问自己：**"如果这件事彻底失败了，我能承受吗？"**

- 探索AI半年找不到工作？ → 存款够撑2年，能承受
- 回东北小县城？ → 能，但我不想
- 继续投Java岗位？ → 能找到，但不是最优解

### 2. 长期视角

**"半年后的我会怎么看这个决定？"**

- 如果我现在随便找份Java工作，半年后可能还是不满意
- 如果我花3个月探索AI，半年后的选择面可能更大

### 3. 价值观对齐

这个决定是否符合我的[[价值观]]？

- 务实 → 不冒险All in
- 自立 → 不依赖别人安排
- 抗脆弱 → 从冲击中成长
- 长期主义 → 愿意短期难受换长期收益

### 4. 信息收集优先

重大决定之前，先收集信息：
- AI领域的机会到底有多少？
- Java + AI的组合岗位存在吗？
- 独立开发的可行性？
- 回老家的成本和生活质量？

## 实际案例：被裁员后的方向选择

| 选项 | 优点 | 缺点 | 最坏情况 | 我的评估 |
|------|------|------|----------|----------|
| A. 继续找Java工作 | 熟练、快上手 | 可能过几年又被裁 | 收入下降 | 保底选项 |
| B. 转型AI相关 | 新机会、有前景 | 学习成本高 | 半年没收入 | 首选探索 |
| C. 独立开发 | 自由、潜力大 | 不稳定 | 存款花完 | 备选方案 |
| D. 回老家 | 成本低 | 机会少 | 失去城市资源 | 最后选项 |

**当前选择：先探索B，同时准备A作为保底。**
